### 2. 诡计设计

#### 密室诡计
```markdown
## 密室类型

### 物理密室
**门窗机关型**：
- 门锁从内部反锁
- 窗户插销锁死
- 利用机关远程锁门

**时间差型**：
- 凶手在发现前离开
- 伪造"不可能离开"的假象

**人为制造型**：
- 故意破门而入
- 凶手就在破门者中

### 心理密室
**错觉型**：
- 死亡时间错觉
- 死亡地点错觉
- 凶手身份错觉

**盲点型**：
- 利用证人盲区
- 利用常识盲点

## 密室设计要点
1. 密室必须成立（不能有漏洞）
2. 解答必须合理（不能过于牵强）
3. 线索必须公平（给读者机会）
4. 诡计必须新颖（避免老套）

## 经典密室示例
**雪地密室**：
- 现象：雪地上只有受害者脚印
- 解答：凶手踩着受害者脚印离开/从空中离开/雪后作案

**电梯密室**：
- 现象：电梯内只有死者
- 解答：利用电梯运行间隙/从天花板出入
```

### 时间诡计

### 视角诡计

#### 线索布置技巧
```markdown
## 公平原则
- 关键线索必须呈现给读者
- 不能隐藏决定性信息
- 线索出现要自然

## 误导技巧
**红鲱鱼（误导线索）**：
- 指向错误嫌疑人
- 制造虚假动机
- 伪造证据

**转移注意**：
- 用显眼线索掩盖关键线索
- 信息过载
- 时间压力

## 伏笔技巧
**淡化处理**：
- 一笔带过
- 夹杂在无关描写中
- 以对话形式提及

**多重含义**：
- 一条线索多重解读
- 表面含义与深层含义

## 线索密度
- 开头：少量，建立基调
- 中段：密集，构建谜团
- 结尾：回收，揭示真相
```

### 7. 推理展示

#### 演绎法
```markdown
## 从一般到特殊
1. 建立一般性原则
2. 观察具体现象
3. 得出特定结论

## 示例
**原则**：所有进入密室的人都会留下痕迹
**观察**：密室内没有第三人痕迹
**结论**：凶手没有进入密室（远程作案/伪密室）
```

#### 归纳法
```markdown
## 从特殊到一般
1. 收集多个具体事实
2. 寻找共同规律
3. 总结一般性结论

## 示例
**事实1**：受害者A被发现时拿着红玫瑰
**事实2**：受害者B身旁有白玫瑰
**事实3**：受害者C的房间有黄玫瑰
**结论**：凶手用玫瑰颜色来标记（或传递信息）
```

#### 消去法
```markdown
## 排除不可能
1. 列出所有可能性
2. 逐一验证排除
3. 剩下的即是答案

## 示例
嫌疑人：A、B、C、D、E
- A有不在场证明 → 排除
- B没有作案能力 → 排除
- C没有动机且证据不符 → 排除
- D的证词有漏洞但可解释 → 保留
- E完全符合所有条件 → 最可疑
```

#### 推理展示技巧
```markdown
## 展示方式
**独白式**：
- 侦探内心推理
- 展示思维过程

**对话式**：
- 侦探与助手对话
- 问答引导

**集中揭示**：
- 召集所有人
- 逐一分析
- 最终指认

## 节奏控制
- 先抛出结论
- 再展示证据
- 层层递进
- 最后收网
```
